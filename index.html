Machine Learning & Regression Analysis

YouTube Video Views Prediction using Machine Learning

An end-to-end Machine Learning pipeline utilizing text modeling and statistical attributes to forecast expected video view counts prior to publishing.

Problem Statement

Content creators lack tools to forecast how their video metadata (like titles) and initial engagement rates (like expected likes) will impact overall view performance. Predicting views beforehand helps content curators optimize titles and plan publication timings more effectively.

Solution Overview

Developed an analytical machine learning regression model. The system processes textual features of video titles using TF-IDF vectorization and merges them with numerical engagement estimators (likes). A tuned XGBoost regressor then predicts overall views, achieving strong prediction accuracy.

Individual Contribution

  • Built the core Machine Learning model to predict video views using title and likes inputs.
  • Performed data preprocessing, feature engineering, and robust text handling for title-based inputs.
  • Applied TF-IDF text vectorization to transform title words into numerical feature weights.
  • Implemented and optimized an XGBoost regression model, achieving a solid prediction performance rating (81% accuracy).

Challenges & Learnings

01
Title Feature Engineering

Challenge: Raw video titles are highly arbitrary and contain non-standard punctuation and emojis, introducing noisy dimensions.

Solution: Deployed specific regex title cleaners, parsed emoji weights, and set max features limits on TF-IDF representation to filter out outlier text weights.

02
Like Count Covariance Bias

Challenge: Model relied too heavily on like numbers to guess view counts, resulting in erratic predictions for new videos with sparse initial likes.

Solution: Introduced L1/L2 regularization criteria inside XGBoost, forcing the model to weight text elements and likes more evenly.

03
Optimizing Regressor Performance

Challenge: Achieving target correlation thresholds under default hyperparameter constraints.

Solution: Conducted grid searches over learning rates, sub-sample ratios, and depth layers, raising test accuracy scores to 81%.

Computer Vision & Object Detection

Detecting Humans in Search and Rescue

Real-time object classification framework engineered to detect human survivors in disaster environments and route notifications to field rescue units.

Problem Statement

Searching for survivors after natural disasters manually over vast camera feeds delays localized help, risking lives during the vital early rescue hours.

Solution Overview

Designed a unified solution that runs YOLO inference on video capture frames. If a human object is classified, coordinate details and image logs are updated in a MySQL db and routed through Django web gateways as mobile alerts.

Individual Contribution

  • Optimized model performance parameters, threshold filters, and frame scaling to ensure faster and accurate detection rates.
  • Configured Django backend services to log detection instances and host the administrative map console.
  • Participated in mobile integration tests ensuring spatial markers sync accurately.

Challenges & Learnings

01
Improving Detection Accuracy

Challenge: Initial testing experienced false positives on objects under low-contrast conditions (dust, shadows).

Solution: Tuned YOLO confidence thresholds and optimized input frame contrast via OpenCV histogram equalization before submitting frames to inference.

02
Handling Real-Time Video Latency

Challenge: Processing sequential frames locked the main execution thread, causing severe UI delay and lag.

Solution: Deployed a producer-consumer thread architecture. The capture stream runs on a separate worker thread feeding a queue, decoupling visual processing from ingestion.

03
Web & Mobile Module Integration

Challenge: Ensuring secure, real-time alert routing between Django and the Android client without battery depletion.

Solution: Established a lightweight JSON-over-HTTP polling sync mechanism with dynamic polling intervals based on location change frequency.
