Real-Time Human Detection for Search and Rescue Using YOLO

An automated computer vision surveillance system built to expedite human detection in critical disaster scenarios and coordinate rescue dispatches via active alert logs.

1. Problem Statement

Traditional search and rescue operations face immense logistical delays due to manual searching in vast disaster environments. Fatigued rescue workers must parse hours of camera feeds manually, leading to oversight and slow response times. Reducing the turnaround time from initial visual sighting to local rescue unit dispatch is a critical factor in saving lives. There is an urgent need for an automated, fast, and highly accurate human detection system that alerts rescue teams in real-time.

2. Solution Overview

Engineered an AI-powered human detection framework combining high-speed computer vision model inference with active web and mobile notifier pipelines. The system processes live camera capture streams, routes frames through a localized YOLO detection engine, validates human sightings, and stores detection metadata in a MySQL database. Django web backend administrators track survivor coordinates and dispatch alerts directly to rescue teams on the ground through a integrated mobile app interface.

Real-time Human Detection

Sub-second frame processing with YOLO to isolate human coordinates.

Automated Alerts

Instant alert notifications generated upon positive validation of survivor status.

Rescue Management

Consolidated administrative dashboard for dispatching rescue teams.

Mobile Sync Integration

Mobile platform interface mapping coordinates with Google Maps API.

6. Challenges & Learnings

01
Contrast & False Positives

Challenge: Processing frames from environments with thick smoke or shadows resulted in missing survivors or mislabeling debris.

Solution: Incorporated pre-inference frame contrast enhancement (contrast stretching via OpenCV) and tuned YOLO confidence thresholds to a balanced 0.6.

02
Ingestion Stream Latency

Challenge: Running YOLO object inference sequentially on the UI thread locked the camera playback frame rate.

Solution: Restructured code into a producer-consumer thread queue model, separating video ingestion threads from the detection model execution block.

7. Impact & Results

The implemented AI model provides a robust proof-of-concept for search and rescue monitoring centers:

  • Improved Detection Efficiency: Replaced continuous manual visual monitoring with automated alerts.
  • Reduced Response Lag: Alerts are routed automatically within seconds of model identification.
  • Automated Incident Logging: Historical search runs are documented automatically in relational SQL blocks for debriefing audits.

Automated Payroll Management System

A web-based administration application built to automate organizational salary distribution and tax calculations while providing secure self-service modules.

1. Problem Statement

Manual payroll calculation is highly prone to human error, resulting in incorrect financial audits, delayed salary distribution, and compliance issues with local tax rules. Traditional spreadsheet-based ledger systems are tedious, difficult to scale, and fail to provide employees with immediate transparency regarding components like gross salary, deductions, and tax exemptions. Moreover, distributing sensitive financial details on insecure sheets exposes organizations to data leaks.

2. Solution Overview

Created a secure, web-based administrative portal engineered to automate ledger distribution, compute gross/net salaries based on pro-rated attendance data, and host employee self-service facilities. Admin users can generate monthly payroll logs, update baseline salary brackets, and review operational metrics. Employees can log in securely using password credentials to access historical invoices, download payslips, and review individual profiles.

Employee Records

Centralized ledger mapping base compensation details and personal information.

Salary & Tax Math

Automated script computations for allowances, deductions, and tax brackets.

Secure Authentication

Encrypted user sessions preventing administrative route leaks.

Reporting Dashboard

Exportable salary statements and payslips generated on request.

6. Challenges & Learnings

01
Dynamic Calculation Consistency

Challenge: Mathematical pro-rating calculations could cause rounding errors over large records during database logs.

Solution: Encapsulated calculation formulas inside dedicated math libraries using fixed-point decimals (`decimal` module) rather than float values.

02
Relational Normalization

Challenge: Preventing data redundancy when employee details (like updates to basic pay) conflicted with historical payouts.

Solution: Structured database schemas to decouple active profile state tables from frozen payout history logs.

7. Impact & Results

Automated HR pipelines provide measurable operational improvements:

  • Reduced Administrative Overhead: Payroll processing is automated with a single button click.
  • Eliminated Calculation Errors: Implemented static formula rules to ensure accurate tax computations.
  • Improved Security: Restricted employee details to verified user account views via route middleware.
